Box-All Storage Management

Box-All Storage in Edmonton is a portable storage bin company. They needed a new system to manage their box inventory, customers, deliveries and invoicing. I had some prior exposure to Ruby, and decided to use Rails for this project. Initially I hosted on Heroku for ease of use, but cost and flexibility led me to use Amazon's Elastic Beanstalk for production in the end. Amazon RDS gave me relatively robust Postgres hosting with backups (I also pull separate backups for Box-All regularly) as well.

I went back and forth regularly with the Box-All manager to get his feedback on the software until we arrived at something that we were both happy with. I enjoy working directly with someone who uses my work to make life easier, it's very satisfying.

So, here we go. This is the login screen (complete with password reset emails and remembered sessions): As you can see I am using Bootstrap for layout. A simple, clean interface was the main requirement here.

After logging in: On this page we can see how many boxes are in use at a glance, todays deliveries which can be printed for the driver (or any other date by clicking the date), and currently active contracts. These screenshots are all local with fake customer data of course.

Here is a customer view page, where we can see their info a glance, as well as create new contracts or edit customer data:

Clicking new contract: We can select the start date with jQuery Datepicker, and other options.

At the bottom of the page we can select from any containers not currently in use:

Here is a newly created contract overview:

Deliveries are scheduled on contracts. They show up on the home page as above, and also on the deliveries screen: Each delivery shows the customer last name and number of boxes being delivered. Zero boxes on a delivery is allowed because Box-All often schedules deliveries before they know what boxes will be delivered. Deliveries can be clicked on for more details (such as destination and notes).

Here we can see an overview of who has what. Box-All management specifically requested this feature.

Invoices are also generated on contracts:

The Invoices index page: Key features here include: -unpaid invoices are marked in red -search invoices by month (clicking on 'Search Invoice Month' drops down a jQuery month selector) -totals button shows monthly gross invoice totals

Some other features not highlighted so far: Search for customers (name or phone), boxes, contracts. Print friendly views of contracts and daily deliveries where paper is still necessary. All views are mobile friendly.

All in all a fairly straightforward application, but I'm happy to say that I built it entirely myself and it is currently used in production at Box-All.
